Reciprocal Intelligence Framework

A proposed governance and technical standard for enforcing community authority over artificial intelligence and digital systems.

Concept only No active certifications

How to use this standard

For Communities & Governance

Use as a checklist for negotiation. If an outside project cannot meet these reciprocity and sovereignty standards, consent should be withheld.

For Researchers & Funders

Use as conditions in MoUs and RFPs. Ensure funding includes budget for independent community governance and offline infrastructure.

Compliance Checklist

  • ☑ Operate under a legitimate Community AI Council with binding powers.
  • ☑ Use FPIC vivo (continuous consent) for all data decisions.
  • ☑ Run on community-controlled, offline-capable infrastructure.
  • ☑ Provide a community-held hard kill switch.
  • ☑ Enforce reciprocity via Value Return Agreements.
  • ☑ Implement Boundary Cards in policy and code.

Implementation Roadmap

Phase 1: Foundations (Current)

Documenting Andean governance models, defining Boundary Cards, and prototyping offline infrastructure reference kits.

Phase 2: Accountable Local Agents

Conditional on community invitation: Co-designing Councils and lab-testing modules under simulated governance reviews.

Phase 3: Federated Network

Aspirational: If territories align, publishing open compliance tests and federated reciprocity protocols.

Reference Frameworks

  • CARE Principles: Collective Benefit, Authority to Control, Responsibility, Ethics.
  • UNDRIP / ILO 169: Standards on self-determination and FPIC.
  • UNESCO AI Ethics: Alignment with global impact assessment tools.
  • Participatory AI: Drawing from democratic governance research (e.g., cip.org, pro-ethics.eu).
