Status and Scope

This document describes a technical module within the Reciprocal Intelligence framework: an SMS-based service for community alerts.

Status: Lab prototype. No field deployments. No community data is currently processed.

Usage: Requires independent FPIC approvals and permits before any real-world testing.

Executive Summary

Reciprocal Intelligence SMS addresses information gaps in rural areas (health post availability, fair market prices, assembly summons) without requiring internet access or smartphones.

Unlike standard broadcast tools, this system enforces community governance logic at the code level: messages can only be sent if valid consent tokens exist, and "STOP" commands trigger verifiable data wipes.

Design Commitments

  • Offline-First: Runs on local hardware (e.g., Raspberry Pi + GSM), not the cloud.
  • Privacy by Design: Phone numbers are stored as salted hashes. Plaintext is kept in an offline "phonebook" file held by a human steward.
  • Collective Consent: The system checks for a community_consent = granted state before allowing individual subscriptions.

Protocol & Architecture

The system uses a simple Command Line Interface (CLI) for stewards:

reciprocal-intelligence broadcast --topic salud --text "Posta medica abierta hasta las 4pm"

Subscription Flow: Users SMS keywords like SALUD or PRECIO. The system validates consent, hashes the number, and returns a unique 4-digit audit code.

Risks & Mitigations

  • Hardware Theft: Mitigated by disk encryption and minimal data retention.
  • Steward Misuse: Mitigated by rotation of credentials and local community oversight (assembly review of logs).

Role of Qori Labs

Qori Labs provides the open-source code (MIT License) and initial reference hardware designs. We do not operate the system, do not access the data, and do not charge for the software. Local communities or partner institutions must operate and govern any deployment.
